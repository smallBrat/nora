# Nora environment variable reference

> Complete reference for every environment variable Nora reads, grouped by category, with required flags, defaults, and configuration guidance.

Every Nora deployment is configured through environment variables. Start by copying `.env.example` to `.env` in your project root, then replace all `<REPLACE_...>` placeholders with real values before starting the stack. The sections below describe every variable Nora reads, grouped by function.

<Note>
  Generate cryptographic secrets with the following command. Never reuse secrets across deployments or commit them to version control.

  ```bash theme={null}
  node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
  ```
</Note>

## Required

These variables must be set before the stack will start. Without them, JWTs cannot be signed, stored credentials cannot be encrypted, backup archives cannot be sealed, and Agent Hub source-catalog API keys cannot be hashed.

| Variable                             | Required | Default | Description                                                                                                                                                                                                |
| ------------------------------------ | -------- | ------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `JWT_SECRET`                         | Yes      | —       | Secret used to sign all JWTs. Minimum 32 characters.                                                                                                                                                       |
| `ENCRYPTION_KEY`                     | Yes      | —       | 32-byte hex key (64 hex characters) for AES-256-GCM encryption of stored provider keys, integration credentials, and other sensitive values.                                                               |
| `NORA_BACKUP_ENCRYPTION_KEY`         | Yes      | —       | 32-byte hex key for sealing managed backup archives. Required to create or restore backups. Do not rotate casually — archives encrypted with the previous key cannot be decrypted with a new one.          |
| `NORA_AGENT_HUB_API_KEY_HASH_SECRET` | Yes      | —       | HMAC secret for hashing Agent Hub source-catalog API keys at rest. `setup.sh` auto-generates this on first run; rotating it invalidates every issued Agent Hub key.                                        |

## Bootstrap admin account

These two optional variables seed a first administrator account on the very first boot. They are ignored on subsequent starts once the account exists. Both must be set together — setting only one has no effect.

| Variable                 | Required | Default | Description                                                                                         |
| ------------------------ | -------- | ------- | --------------------------------------------------------------------------------------------------- |
| `DEFAULT_ADMIN_EMAIL`    | No       | —       | Email address for the bootstrap admin account.                                                      |
| `DEFAULT_ADMIN_PASSWORD` | No       | —       | Password for the bootstrap admin account. Minimum 12 characters. No insecure defaults are accepted. |

<Warning>
  Do not set a weak or well-known password here. This account has full operator access from the moment the database is seeded.
</Warning>

## Database

| Variable      | Required | Default    | Description                                                                  |
| ------------- | -------- | ---------- | ---------------------------------------------------------------------------- |

| `DB_PORT`     | No       | `5432`     | PostgreSQL port.                                                             |
| `DB_USER`     | No       | `nora`     | Database role used by backend-api and worker-provisioner.                    |

| Variable                          | Required | Default | Description                                                                                            |

| `GOOGLE_CLIENT_SECRET`            | No       | —       | OAuth 2.0 client secret from Google Cloud Console.                                                     |

## Runtime selection

Runtime selection is three-dimensional: **runtime family × deploy target × sandbox profile**. Each enabled-list variable accepts a comma-separated list. The first id in each list is the default for new deployments. See [Provisioner backends](/configuration/provisioner-backends) for details.

| Variable                   | Required | Default    | Description                                                                                                                  |
| -------------------------- | -------- | ---------- | ---------------------------------------------------------------------------------------------------------------------------- |
| `ENABLED_RUNTIME_FAMILIES` | No       | `openclaw` | Runtime families enabled. Supported ids: `openclaw`, `hermes`.                                                               |
| `ENABLED_BACKENDS`         | No       | `docker`   | Deploy targets enabled. Supported ids: `docker`, `k3s`, `k8s`, `proxmox`. `k3s` and `k8s` share the same Kubernetes adapter. |
| `ENABLED_SANDBOX_PROFILES` | No       | `standard` | Sandbox profiles enabled. Supported ids: `standard`, `nemoclaw`.                                                             |

### K3s and cloud Kubernetes Compose overlays

`docker-compose.k3s.yml`, `docker-compose.aks.yml`, `docker-compose.gke.yml`, and `docker-compose.eks.yml` are examples for running the Nora control plane in Compose while provisioning agents into Kubernetes. These variables are Compose helper inputs; the overlays translate them into `KUBECONFIG` and `K8S_*` inside `backend-api` and `worker-provisioner`.

| Variable pattern | Description |
| ---------------- | ----------- |
| `<PROVIDER>_KUBECONFIG_PATH` | Host path to the kubeconfig mounted at `/kubeconfig/config` inside Nora containers. Providers: `K3S`, `AKS`, `GKE`, `EKS`. |
| `<PROVIDER>_ENABLED_BACKENDS` | Enabled deploy targets for that overlay. K3s defaults to `k3s`; cloud overlays default to `k8s`. |
| `<PROVIDER>_K8S_NAMESPACE` | Namespace used for agent workloads. Defaults to `openclaw-agents`. |
| `<PROVIDER>_K8S_EXPOSURE_MODE` | Exposure mode passed to `K8S_EXPOSURE_MODE`. K3s defaults to `node-port`; managed-cloud overlays default to `load-balancer`. |
| `<PROVIDER>_K8S_RUNTIME_HOST` | Hostname or IP Nora uses for NodePort runtime and gateway access. Used by the K3s overlay. |
| `<PROVIDER>_K8S_RUNTIME_NODE_PORT` | Optional fixed runtime NodePort. Leave empty for dynamic per-agent allocation. Used by the K3s overlay. |
| `<PROVIDER>_K8S_GATEWAY_NODE_PORT` | Optional fixed gateway NodePort. Leave empty for dynamic per-agent allocation. Used by the K3s overlay. |
| `<PROVIDER>_K8S_SERVICE_ANNOTATIONS_JSON` | JSON object of provider-specific Service annotations. |
| `<PROVIDER>_K8S_LOAD_BALANCER_SOURCE_RANGES` | Comma-separated CIDRs allowed to reach the per-agent cloud load balancers. |
| `<PROVIDER>_K8S_LOAD_BALANCER_CLASS` | Provider-specific `loadBalancerClass`, such as `networking.gke.io/l4-regional-external` or `eks.amazonaws.com/nlb`. |
| `<PROVIDER>_K8S_LOAD_BALANCER_READY_TIMEOUT_MS` | Load-balancer readiness timeout. Defaults to `600000` for K3s and `1200000` in the cloud overlays. |
| `<PROVIDER>_K8S_LOAD_BALANCER_READY_INTERVAL_MS` | Load-balancer readiness polling interval. Defaults to `5000`. |
| `<PROVIDER>_DEPLOYMENT_JOB_TIMEOUT_MS` | Deployment queue job timeout. Defaults to `900000` for K3s and `1800000` in the cloud overlays. |
| `<PROVIDER>_PROVISION_TIMEOUT_MS` | Worker-side provision timeout. Defaults to `840000` for K3s and `1740000` in the cloud overlays. |

### SSH / SFTP

Use `NORA_BACKUP_STORAGE=ssh`.

| Variable                      | Required | Default | Description                                        |
| ----------------------------- | -------- | ------- | -------------------------------------------------- |
| `NORA_BACKUP_SSH_HOST`        | No       | —       | SSH host.                                          |
| `NORA_BACKUP_SSH_PORT`        | No       | `22`    | SSH port.                                          |
| `NORA_BACKUP_SSH_USERNAME`    | No       | —       | SSH user.                                          |
| `NORA_BACKUP_SSH_REMOTE_PATH` | No       | —       | Remote path where backup archives are uploaded.    |
| `NORA_BACKUP_SSH_PRIVATE_KEY` | No       | —       | PEM-encoded private key for SSH auth.              |
| `NORA_BACKUP_SSH_PASSWORD`    | No       | —       | Password fallback when key auth is not configured. |

## LLM key storage

| Variable      | Required | Default    | Description                                                                                                                                   |
| ------------- | -------- | ---------- | --------------------------------------------------------------------------------------------------------------------------------------------- |
| `KEY_STORAGE` | No       | `database` | Where LLM provider keys are stored. `database` encrypts and persists keys in PostgreSQL. `env` reads keys from environment variables instead. |

| Variable        | Required | Default | Description                                                                                   |

| `TLS_CERT_PATH` | No       | —       | Path to the TLS certificate file on the host. Used when nginx is configured to terminate TLS. |
| `TLS_KEY_PATH`  | No       | —       | Path to the TLS private key file on the host.                                                 |

## Cost reporting

These optional rates drive the Cost panels on agent detail pages. Leave unset to hide cost figures.

| Variable               | Required | Default | Description                                                               |
| ---------------------- | -------- | ------- | ------------------------------------------------------------------------- |
| `COST_PER_VCPU_HOUR`   | No       | —       | Cost in your currency per vCPU per hour. Used for compute cost rollups.   |
| `COST_PER_GB_RAM_HOUR` | No       | —       | Cost per GB of RAM per hour.                                              |
| `COST_PER_1K_TOKENS`   | No       | —       | Fallback LLM token cost when a provider's per-model rate isn't available. |
