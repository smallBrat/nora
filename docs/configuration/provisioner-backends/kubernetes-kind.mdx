# Kind Kubernetes provisioner backend

> Verify Nora's Kubernetes provisioner locally with Kind and NodePort Services.

Kind is the local smoke-test path for the Kubernetes adapter. It is not intended to model cloud load balancer behavior.

## Automated smoke

Run the built-in smoke script:

```bash theme={null}
cd e2e
npm run smoke:k8s-kind
```

The script creates or reuses a Kind cluster, exports kubeconfigs, starts the Nora API and provisioner with `docker-compose.kind.yml`, deploys an agent, verifies Kubernetes objects, and tests stop/start/restart lifecycle.

## Manual stack

The Kind overlay configures Nora for NodePort exposure:

```bash theme={null}
docker compose -f docker-compose.yml -f docker-compose.kind.yml up -d --build
```

The overlay sets:

```bash theme={null}
ENABLED_BACKENDS=k8s
KUBECONFIG=/kubeconfig/config
K8S_EXPOSURE_MODE=node-port
K8S_RUNTIME_NODE_PORT=30909
K8S_GATEWAY_NODE_PORT=31879
K8S_RUNTIME_HOST=host.docker.internal
```

## Troubleshooting

Check the Kubernetes workload:

```bash theme={null}
kubectl -n openclaw-agents get deploy,svc,pods
```

Check Nora worker logs:

```bash theme={null}
docker compose -f docker-compose.yml -f docker-compose.kind.yml logs worker-provisioner
```

If the agent reaches `running` but gateway embed is not reachable, confirm the NodePort values match the Kind cluster port mappings in `infra/kind/nora-kind.yaml`.
