# Per-provider integration guides

> Where to apply for credentials, which scopes to request, and how to wire each integration into Nora.

The main [integrations guide](/guides/integrations) covers the connection workflow that every provider follows — opening the integration modal, pasting credentials, and syncing to the running agent. This index links to one page per provider with the bits that differ between platforms: where the developer console lives, the exact scopes Nora expects, and any gotchas.

## How to use these guides

1. Find the provider you want to connect in the catalog below.
2. Open the linked guide and follow the **Where to apply for credentials** section.
3. Copy the credentials into the Nora dashboard's integration modal — the modal also surfaces the `credentialsUrl` and setup steps inline, so you can keep the docs and the dashboard side-by-side.
4. For OAuth providers, register the redirect URI shown in the modal **before** clicking _Authorize_.

<Note>
  Many providers also publish a Model Context Protocol (MCP) server. Where one exists, the
  per-provider page lists the npm/PyPI package and a link to its documentation. Nora's catalog
  surfaces this as an "MCP" badge on the integration card, but the runtime does not auto-launch the
  MCP server today — you bring it up alongside the integration if you want it.
</Note>

## Per-provider page template

Each provider page follows the same outline so you always know where to look:

1. **Where to apply for credentials** — the developer console URL and the exact UI path.
2. **Required scopes / permissions** — what the credential needs to grant.
3. **Connect in Nora** — what to paste into the modal.
4. **Verify the connection** — what `Test connection` calls under the hood.
5. **MCP server** (if one exists) — package name + link.
6. **Troubleshooting** — common errors and what they mean.
